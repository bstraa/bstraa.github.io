Web Scraping and Regression Analysis

Posted on Sat 28 February 2015 in Python

I recently did a data science project where I predicted movie revenue (total adjusted domestic gross). It has been a valuable experience because there is a lot to learn by doing a complete project from start to finish, especially when acquiring and cleaning your own data. It also amplifies the importance of design. Past Ben definitely learned some valuable lessons here - it is certainly true that 'hindsight is always 20/20'!

Background

I used the Python library Beautiful Soup to acquire data from Box Office Mojo and conducted data analysis using Pandas before doing some modelling with scikit-learn and statsmodels.

I imagined being a company interested in predicting total adjusted domestic gross before a movie is released because it could be extremely valuable if selling products associated with a particular movie.

Initial Exploration

Box Office Mojo has over 16,000 movies available with an impressive amount of data, but the layout is fairly unintuitive. It took some time exploring to figure out the optimal strategy to scrape pages. After poking around a bit, I found an alphabetical directory which contains links to individual movie pages with quite a bit of information. I decided to scrape the links and any additional information pertaining to the corresponding movies.

These links allowed me navigate through even more pages, where the true gems were located. There was a lot of data I wasn't sure would be useful, but figured I'd rather be safe than sorry. Here's a screen shot (MAN, this movie brings me back!):

Cleaning and Familarizing

After scraping and loading my data into Pandas, there were several issues which emerged before I could get going on analysis:

  • NA values

  • Removing dollar signs, etc.

  • Converting strings to datetime objects.

  • Converting strings (typically measurements) to integers.

After these steps, I was pretty excited to get going with further analysis and modelling. Whoever said much of data science is not modelling was not joking!

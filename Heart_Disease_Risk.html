Heart Disease Risk

Posted on Thu 03 March 2016 in Python

Background

At Metis a group of us used the UCI Machine Learning Repository to create a product which would help individuals assess their risk for heart disease without having to see a doctor. Given user input from our web app (created using Flask), users would receive a heart disease risk score visualized using D3.js. Below is a snippet of our custom gauge inspired by this.

I really enjoyed the process of building something extremely useful - how cool is it that someone can find their risk of heart disease from home, with information almost everyone has!? (We created a few different models - from using simple variables everyone knows to less known ones). I was elected to 'product manager' which I really enjoyed - it is pleasurable to work with groups of intelligent people who think differently to build something.

This project wasn't as cleaning intensive as this one I did, so I wanted to spend some time really digging into the various classification algorithms and really understanding the mathematical framework behind them and assess their practical use (as data science is bound by computer science) in the workplace.
